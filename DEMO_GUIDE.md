# 智能会议助手演示指南

## 快速开始演示

### 1. 启动会议

```bash
# 直接在浏览器中打开主页面
start index.html
```

或者：
```bash
# 使用Python启动本地服务器（推荐）
python -m http.server 8000
# 然后在浏览器访问 http://localhost:8000
```

### 2. 模拟参会人员提交内容

1. **方式一：直接使用浏览器访问**
```bash
# 在新标签页打开输入页面，替换会议ID
start "input.html?meeting=MEET-20260331-ABC123"
```

2. **方式二：使用二维码扫码**
- 用手机扫描页面上的二维码
- 手机会自动打开输入页面

### 3. 测试功能

#### 文字输入测试
- 在输入页面填写姓名
- 输入会议内容，例如：
  ```
  工作进展：已完成用户管理模块开发，正在测试API接口。
  需要协助：数据库查询性能优化需要帮助。
  ```
- 点击"提交内容"按钮

#### 语音输入测试
- 点击麦克风按钮开始录音
- 等待3秒后会自动生成模拟的语音识别结果
- 可以多次录制添加内容

#### 模板输入测试
- 点击底部的快速模板按钮
- 选择合适的模板自动填充内容

### 4. 会议分析

1. 在主页面点击"分析会议内容"按钮
2. 等待2秒后会显示AI分析结果
3. 查看：
   - 会议总结
   - 讨论重点
   - 行动项
   - 参与人员

### 5. 历史功能测试

1. **查看历史会议**
   - 点击"查看历史会议"按钮
   - 选择任意历史会议查看详情

2. **待办事项管理**
   - 点击"待办事项回顾"按钮
   - 标记待办事项为已完成
   - 查看已完成的事项

## 技术验证

### 1. 二维码验证
- 扫描二维码是否能正确打开输入页面
- 会议ID是否正确传递
- 刷新二维码是否生成新的会议ID

### 2. 数据存储验证
- 提交内容后localStorage是否有数据
- 历史会议是否正确保存
- 待办事项是否可以标记完成

### 3. 界面响应验证
- 调整浏览器窗口大小，布局是否自适应
- 在不同设备上测试显示效果
- 动画是否流畅自然

## 常见问题排查

### 1. 二维码不显示
- 检查网络连接，确保能加载QRCode.js
- 查看浏览器控制台是否有错误
- 尝试刷新页面

### 2. 提交内容不显示
- 检查浏览器localStorage是否正常工作
- 确认会议ID是否一致
- 查看主页面实时统计是否更新

### 3. AI分析无结果
- 确保至少有一条会议内容
- 检查浏览器控制台是否有错误
- 尝试重新提交内容后再分析
